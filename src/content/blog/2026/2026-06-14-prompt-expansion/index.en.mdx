If you use LLMs, probably you hear about the importance of a good prompt to get a good result. This is an "old" concept: "prompt engineering" concept, it was hype in 2023.

But still have meaning, not the same importance as 3 years ago, when everything lay on the prompt, and still being important, but now we have other techniques to improve or enrich the prompt the LLMs receives, for example **prompt augmentation** (or enrichment) which gets your prompt and add it: context, structural rules, domain knowledge, etc.

But the goal of this post, is just undertand the basics and to show you a very simple, but useful, technique to get better results for one shot prompts. 


# What is prompt expansion?

Prompt expansion is an AI technique that uses an LLM to generate prompts which are: 

- detailed
- less ambiguous
- complete


In other words, prompts generated by a LLM from a human (or no) prompt, that are more complete for the task to execute than the original one.


Lets see it with an example: You want to init or bootstrap a frontend project, there are a lot of small things to do and to decide: The tech stack, the linter, the linter's riles, the formater, the package manager, the runtime, typescript or not, testing tools and testing strategy, etc. A lot but probably it is not the first time you bootstrap a similar project, so instead of do a detailed list of thast you can create a simple prompt:

> "I want to create a prompt to bootstrap a Home Assistant frontend card addon. Use https://github.com/sergiocarracedo/sc-custom-cards as base for the tech stack and the coding decisions. We should ask the user for the addon name and for the cards to create.
> 
> Ask me any gap or doubt you have to generate a very detailed description of the tasks to do and ask to the user for this bootstrap and provide a check list of the expected outcome"


This part is very important "**Ask me any gap or doubt ...**", it forces the LLM to not provide the final prompt infering or hallucinating the missing parts and ask you for them

:::codeblock{maxHeight=300}

```markdown
│   │   ├── [card-1]/

## 📝 Implementation Details per File
1. package.json
Configure tasks: "dev": "vite build --watch", "build": "vite build", and lint/format scripts.

Dependencies: Include lit and custom-card-helpers.

DevDependencies: Include typescript, vite, vite-plugin-css-in-js (or similar to bundle styles into the JS), and your formatting/linting tooling.

2. vite.config.ts
Configure Vite to build in library mode.

Output a single bundle file named [addon-name].js inside the dist folder.

Ensure minification is turned on for production builds but keeps code readable enough for debugging in dev mode.

3. src/main.ts (The Registry Entrypoint)
Must hook into window.customCards to register every generated card so they appear natively in the Home Assistant visual card picker dashboard.

Example schema to inject:

TypeScript

window.customCards.push({
  type: 'card-1-tag-name',
```

### Tips and tricks

- Ask LLMs to **not fill the gaps assuming things** and ask you: "Ask me any gap or doubt you have"
- **Give it context**: For example a reference: e.g. "I want to generate a prompt do text correction baes in the following text style: [REFERENCE TEXT]"
- Ask LLM to **behave as a specific role** if needed: e.g. "You are a Literature Nobel price winner writter....". It forces the LLM to ask questions and solve problem with domain specific context
- **Read** the results carefully to ensure it fits what you expect and to detect missing elements
- **Don't settle for just one result**. As LLMs are not deterministics every time you ask for a prompt you will get a different output. Generate at leat 2 and get the better parts or ideas from both to create one.

## Image to prompt

Did you see an image you liked and you want to generate your version using an AI? You can just paste the image and tell the LLM the changes you did, but you can also do something smarter, reusable and repetible:

### Describe image style from an image

Imagine you want to create images with the same visual style of "Streamboat Willie" movie (as it is a [public domain](https://curell.com/mickey-mouse-y-derechos-de-autor/) movie now)


:::gallery{cols="3"}
streamboat01.png
streamboat02.png
streamboat03.png
:::

We can as an LLM to generate a base prompt:

> From the images, generate a prompt which distills the visual style of the images, be detailed respect to generic styles, not about the items or specific characters are in the images. In the prompt reserve a placeholder for the user to describe the image it want to generate with this style


In my case the generated prompt was:

:robot: 
```
A black and white traditional hand-drawn 2D animation still, evoking the style of late 1920s cartoons. The aesthetic features high-contrast, grainy film quality with visible scratches and flicker. The artwork consists of thick, wobbly ink lines and simple, rounded shapes. Backgrounds are soft, hand-painted grayscale watercolor washes on textured paper, often with a blurred, distorted perspective. There is no smooth in-betweening; the animation has a jerky, energetic, and slightly imprecise feel, with a noticeable film grain overlay. Characters and objects have a simplistic, rubber-hose quality.

[INSERT YOUR DESCRIPTION OF THE IMAGE TO GENERATE HERE]
```


And now I can store this prompt (without the images, we don't need it anymore), and just use it when we need it, generating images like these:

# Summary

This is an "old", very simple and basic technique to improve the LLMs results, with no efford, but still being important when you work with models.

If you add to the ecuation SKILLs in both ways: skills to help you to generate the prompts you need (for example using [brainstorming](https://www.skills.sh/obra/superpowers/brainstorming)), and the skill generation instead a "classic" prompt, you have a powerful tool.
